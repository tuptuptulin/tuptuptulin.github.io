Understanding Alternate Data Streams (ADS) in NTFS

NTFS, the Windows file system, has a feature called Alternate Data Streams (ADS) that allows attaching hidden data to files. This tutorial explores how to work with ADS using PowerShell.

What are Alternate Data Streams?

Alternate Data Streams (ADS) let you attach additional hidden data to a file without altering its visible size, making it a powerful feature for storing metadata or securing information.

Practical Example of ADS

Let's go through a PowerShell example where we create a main file and attach different streams to it.

  • Create a file: Set-Content -Value "demoText" -Path "demo.txt"
  • Add a stream: Set-Content -Path ".\demo.txt" -Value "Stream1 text" -Stream "Stream1"

Read the content of each stream with Get-Content .\demo.txt -Stream 'Stream1' to view the hidden data.

Conclusion

ADS can be used to store additional data within files, providing security, metadata storage, and advanced file management options for NTFS users.
